# GamersHub Product Context

## What This Platform Is
- A university-focused esports and gaming community platform
- Part esports management tool, part social platform, part livestream discovery surface

## Primary User Journeys
- A student registers, verifies identity context, logs in, and lands on the correct dashboard.
- A player browses tournaments, streams, and the social feed.
- A user creates posts, reacts, comments, and manages friends.
- An admin moderates platform content and reviews operational data.
- A superadmin manages role changes and higher-level oversight tasks.

## Features That Matter Most To The Project
- secure authentication
- MFA
- role-based access control
- player community interaction
- tournament and livestream visibility
- analytics and reporting
- auditability for admin actions

## Decision Filter For Claude
- Choose changes that make the platform more coherent for a university esports workflow.
- Prefer practical deliverables over architecture theater.
- If a change helps satisfy software-lab requirements without breaking current behavior, it is usually a good fit.
- For UI work, prefer extending the current Figma-backed GamersHub design language over inventing new patterns.
